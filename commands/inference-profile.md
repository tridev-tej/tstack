# AWS Bedrock Inference Profile Tool

## User Request
- **Action**: $ARGUMENTS.action
- **ARN/Region**: $ARGUMENTS.arn_or_region

## Instructions

Based on the action requested, help the user with AWS Bedrock inference profiles.

### If action is "get" (default)

Extract the region from the ARN and run:
```bash
aws bedrock get-inference-profile \
  --region <extracted-region> \
  --inference-profile-identifier "<full-arn>"
```

Then explain:
1. **Profile Name**: The human-readable name
2. **Profile Type**: System (cross-region) vs Application (user-created)
3. **Underlying Model(s)**: Extract from the `models` array - this is the actual foundation model
4. **Status**: Whether it's active
5. **Created By**: If available in tags

### If action is "list"

List all inference profiles in the specified region:
```bash
aws bedrock list-inference-profiles --region <region> --type-equals APPLICATION
```

### If action is "explain"

Explain the two types of inference profiles:

| Type | ARN Pattern | Naming | Use Case |
|------|-------------|--------|----------|
| **System/Cross-Region** | `inference-profile/global.anthropic.claude-*` | Auto-generated by AWS, model visible in ID | Route requests across regions for resilience |
| **Application** | `application-inference-profile/<id>` | User-created with custom name, ID is auto-generated | Track costs, apply tags, custom routing |

**Key Points:**
- System profiles: Model name is visible in the ARN (e.g., `global.anthropic.claude-haiku-4-5-20251001-v1:0`)
- Application profiles: Model is hidden - must call `get-inference-profile` to see the underlying model
- Application profiles wrap a foundation model or system inference profile
- Tags on application profiles help with cost allocation and access control
